Die Suchanfrage wird benutzt um eine Menge von Inhalten dynamisch zu bestimmen, z.B. die Top 3 Nachrichten eines bestimmten Bereichs. Als Redakteur müssen Sie dadurch nicht mehr die Inhalte an einem bestimmten Platz innerhalb der Webseite manuell austauschen, da dies durch die gespeicherte Suchanfrage automatisch erfolgt.
Die Suchanfrage kann sowohl Standardsuchanfrageparameter (z.B. "keyword: Sport"), als auch personalisierte Suchfunktionen (z.B. "userKeywords(...)") enthalten. Verknüpfungen mit "AND"/"OR" sind möglich. Weitere Informationen zur verwendbaren Suchanfragesprache finden Sie unter http://wiki.apache.org/solr/SolrQuerySyntax. Welche Felder im Suchindex vorhanden sind, ist abhängig von Ihrer CoreMedia-Installation und den gemachten Anpassungen. Bitte fragen Sie dazu Ihren Systemadministrator bzw. schauen Sie in die Konfiguration der CoreMedia-Searchengine.

Die personalisierten Suchfunktionen erzeugen einen Teil der Suchanfrage dynamisch für einen konkreten Webseitenbenutzer. Zum Beispiel kann die Suchfunktion "userKeywords(...)" die vom aktuellen Benutzer meistgeklickten Keywords in die Suchanfrage einbauen. Dadurch werden nur Inhalte gefunden, deren Inhalte den Interessen des Benutzers entsprechen.

Die folgende Tabelle enhält eine Liste der installierten Suchfunktionen:

Name Beschreibung
contextProperty

Suchfunktion, die den Wert einer Property eines Kontexts zur Suchanfrage hinzufügt.

Parameter:
property
Die Property des Kontexts in der Form <context>.<property>
field
Das Suchmaschinenfeld, das durchsucht werden soll

Beispiel: Angenommen, es gibt eine Property "foo" des Kontexts "bar" mit dem Wert "42". In diesem Fall würde der Aufruf "contextProperty(property:bar.foo, field:field)" zu "field:42" ausgewertet werden.

userKeywords

Suchfunktion, die die Keywords aus dem Kontext eine Benutzers auswählt, die eine Bedingung hinsichtlich der Gewichtung erfüllen.

Parameter:
limit
Beschränkt die Anzahl der zurückgelieferten Keywords (ein negativer oder fehlender Wert heißt "keine Beschränkung")
field
Das Suchmaschinenfeld, das durchsucht werden soll
threshold
Die Mindest-Gewichtung, die die zurückgelieferten Keywords haben sollen
context
Der Kontext, der die Keywords enthält

Beispiel: Angenommen, Sie haben ein Kontextobjekt mit dem Namen "myContext" und den Properties ("foo", 0.8), ("bar", 0.5) und ("zork", 0.1). Dann würde "userKeywords(threshold:0.5, limit:-1, field:field, context:myContext)" durch "field:(foo bar)" und "userKeywords(threshold:0.5, limit:1, field:field, context:myContext)" durch "field:(foo)" ersetzt werden.

userSegments

Suchfunktion, die die Menge von Benutzersegmenten auswählt, zu denen ein angemeldeter Benutzer gehört.

Parameter:
field
Das Suchmaschinenfeld, das durchsucht werden soll
context
Der Kontext, der die Segment-Properties enthält

Beispiel: Angenommen, Sie haben ein Kontextobjekt mit dem Namen "myContext" und den Properties ("content:42", true), ("content:44", false) und ("content:46", true). In diesem Fall würde "userSegments(field:field, context:myContext)" durch "field:(42 46)" ersetzt werden. Diese Funktion ist für die Nutzung mit den Benutzersegmenten von CoreMedia Adaptive Personalization gedacht. Diese nutzt Property-Keys der Form "content:<segmentID>", (wobei segmentId die numerische Content Id des Segments ist) um Segmente in dem Kontext eines Benutzers abzubilden.

userVisitedPages

Suchfunktion, die sich merkt, welche Seiten ein Benutzer bereits betrachtet hat.

Beispiel: Angenommen, Sie möchten eine Liste von Dokumenten anzeigen, die der Benutzer bereits betrachtet hat. Darüber hinaus kann die Suchfunktion benutzt werden, um Elemente aus einer Liste zu filtern, die der Benutzer bereits betrachtet hat(NOT userVisitedPages()).

userSegmentTaxonomies

Eine Suchfunction die überprüft, welche Taxonomien in einem Segment verlinkt sind.

Parameters:
field
the field the taxonomy is linked under

Beispiel: Ein Benutzer kam über eine Suche nach "Sport" auf die Website. Da dies eine Bedingung des Segments Sport ist, wird der Benutzer diesem Segment zu gewiesen. Das Segment beinhaltet als weitere Regel, dass alle Dokumente die mit Sport getaggt sind, ebenfalls diesem Segment zugewiesen werden. Deswegen kann man nun mit Hilfe der Suchfunktion die Verbindung zwischen dem Segment "Sport" und den getaggten Dokumenten herstellen.
